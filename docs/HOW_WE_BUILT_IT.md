# 这个项目是怎么一步步做出来的

> 用「讲给人听」的方式，说说我们怎么从「有一堆图片」做到「自动发到 Pinterest、还能定时发、还能按分区发」。适合讲给不懂技术的同事、老板或朋友。

---

## 一、我们一开始想解决什么问题？

- **手里已经有很多产品图**（比如厨房柜子、收纳柜的图）。
- **希望这些图自动发到 Pinterest**，而不是一张张手动上传、写标题、选画板。
- **还希望**：标题、描述、标签尽量专业（SEO、长尾词），而且最好能**定时发**、发到**对的画板、对的分区**。

所以目标很清晰：**做一个「自动化发 Pinterest Pin」的小工具**，你只管准备好图片，剩下的尽量交给程序。

---

## 二、第一步：先能「登录 Pinterest」并拿到权限

Pinterest 不会随便让任何程序代替你发图，必须先**用你的账号登录一次、授权**，程序才能拿到一张「通行证」（token），之后代你发 Pin。

我们做的事：

1. **建了一个本地小网站**（就跑在你电脑上，不公开），只有一个页面：点一下就会跳转到 Pinterest 的登录页。
2. 你在 Pinterest 上点「同意」之后，Pinterest 会跳回我们的小网站，并送给我们一个「授权码」。
3. 我们用这个授权码去 Pinterest 换一张**长期的通行证**（access token），存到本地的 `tokens.json` 里。

这样，以后每次发 Pin，程序都会带着这张通行证去请求，Pinterest 就知道是「你授权的这个程序在替你操作」。

**对非技术的人可以这么说**：  
「我们先做了一个‘用你的 Pinterest 账号登录一次’的步骤，登录成功后，程序会拿到一张电子通行证，以后发图都靠这张通行证，不用你再登录。」

---

## 三、第二步：能「列出你的画板」并「发一张 Pin」

有了通行证，我们就可以调用 Pinterest 的官方接口（API）做两件事：

1. **列出你账号下所有画板**  
   - 这样你就知道要发到哪个画板，程序也能用「画板名字」自动对号入座。
2. **发一张 Pin**  
   - 程序把一张本地图片转成网络可识别的格式，加上标题、描述、链接等，一次性发给 Pinterest，Pinterest 就会把这张图当成你发的一条 Pin。

一开始我们**不搞复杂**：  
- 你指定一个画板 ID（从「列出画板」那步抄过来）。  
- 程序从某个文件夹（比如 `assets/to-post`）里读图，一张张发，发完的把图移到「已发布」文件夹，失败移到「失败」文件夹。

**对非技术的人可以这么说**：  
「第二步我们让程序能：一、读出你有哪些画板；二、把指定文件夹里的图片，按你选的画板一张张发出去，并且自动把已发和失败的图分到不同文件夹。」

---

## 四、第三步：让程序「自己决定发到哪个画板」——用文件夹名或关键词

手动复制画板 ID 还是麻烦，所以我们加了「按名字匹配」：

- 你告诉程序一个**关键词**（比如「厨房」「Kitchen」），程序在你所有画板里找**名字或描述里包含这个词**的画板，用找到的第一个。
- 或者你把图片放在**以画板名命名的子文件夹**里（比如 `images/厨房`），程序用文件夹名当关键词去匹配画板。

这样你就不用记画板 ID，只要记住画板叫什么就行。

**对非技术的人可以这么说**：  
「后来我们加了一个小功能：你可以只说一个词，比如‘厨房’，程序会自动找到名字里带‘厨房’的那个画板，把图发到那里；或者你把图放在名为‘厨房’的文件夹里，程序也会自动对号入座。」

---

## 五、第四步：用 AI 根据「图片内容」写标题、描述，并选画板

这里我们接入了 **OpenAI**（和 ChatGPT 同一家），让 AI「看图说话」：

- **看一张图**，自动生成：  
  - 一条适合 Pinterest 的**标题**（我们定了格式：风格 + 布局 + 功能 + 用途，并且尽量带长尾关键词）；  
  - **描述**（必须带品牌名 The Cabination，并自然带长尾词）；  
  - **标签**（#xxx）；  
  - **无障碍描述**（alt），方便搜索和无障碍。
- 同时，AI 会给这张图一个**类别词**（比如 kitchen、storage），我们再用这个词去匹配你的画板名字，实现「按图自动选画板」。

你还可以提供一个**长尾关键词清单**（放在 `keywords.txt` 或环境变量里），AI 会尽量在标题、描述、alt 里自然用上这些词，有利于 SEO。

**对非技术的人可以这么说**：  
「我们接入了 AI：程序会‘看’每一张图，自动写出标题、描述、标签，并且根据图的内容自动选一个最合适的画板。我们还可以给 AI 一份‘希望多出现的词’清单，它会尽量用进去，方便被搜索到。」

---

## 六、第五步：支持「画板下面的分区」（Section）

Pinterest 的一个画板下面可以有多个**分区**（Section），比如「Modern Kitchen Cabinets」下面有「Navy Blue」「Modern Gray」等。我们加了：

1. **列出某个画板的所有分区**（命令：`npm run sections -- --board=画板ID`），这样你能看到每个分区的名字和 ID。
2. **发 Pin 时指定分区**：  
   - 要么用分区 ID；  
   - 要么用「分区名称里包含的某个词」（比如 "Navy Blue"），程序自动匹配到对应分区，把 Pin 发到那个分区里。

**对非技术的人可以这么说**：  
「你的一个画板下面可以有好几个小分区。我们让程序能：一、列出这个画板下所有分区；二、发图时可以指定‘发到名字里带某某词的那个分区’，这样图就会自动进到对的分区里。」

---

## 七、第六步：「先预览、人工改一遍再发」——符合平台政策

Pinterest 希望每条 Pin 都是**用户知情、有意发布**的，所以我们加了一个「预览再发」的流程：

1. 程序**只生成**每条 Pin 的标题、描述、分区、alt 等，写进一个 JSON 文件（`pin-preview.json`），**不真正发到 Pinterest**。
2. 你打开这个文件，可以**改文案**，或把某一条标成「不发」（`skip: true`）。
3. 你再跑一条命令（`npm run post-pins -- --from-preview`），程序**只把预览里没被标成「不发」的**发出去。

这样既保留了自动化（AI 写文案、选画板/分区），又保留了人对「发什么」的参与，符合平台对「用户主动选择每条 Pin」的期待。

**对非技术的人可以这么说**：  
「我们加了一个‘先看再发’的步骤：程序先把要发的内容列在一份预览文件里，你可以改字、或勾掉不想发的；确认后再点一下‘发布’，程序才会真正发到 Pinterest，这样既省事又符合平台要求。」

---

## 八、第七步：定时自动发（每天固定时间发一批）

最后我们加了**定时发**：

- 在环境变量里配置：**发到哪个画板**、**每天几点发**（支持 cron 表达式，比如每天 9:00、或每天 9:00 和 15:00）、**是否用 AI 写文案**。
- 运行一个「定时任务」脚本（`npm run schedule`），它会常驻在后台，到点就自动执行「从某个文件夹读图 → 按当前配置发 Pin」。
- 不想常驻的话，也可以用电脑自带的「计划任务」（crontab）到点执行同一条发 Pin 命令。

**对非技术的人可以这么说**：  
「我们加了一个‘定时发’功能：你设定好每天几点、发到哪个画板，程序会在那个时间自动把准备好的图发出去，你只要提前把图放进指定文件夹就行。」

---

## 九、整体流程串起来（给不懂技术的人一句话版）

1. **登录一次**：用你的 Pinterest 账号在本地网页授权，程序拿到通行证。  
2. **准备图**：把要发的图放进「待发」文件夹。  
3. **发 Pin**：  
   - 可以指定画板、分区；  
   - 也可以用 AI 自动写文案、自动选画板/分区；  
   - 还可以先生成预览，你改完再发。  
4. **定时发**（可选）：设定好时间和画板，程序到点自动发「待发」文件夹里的图。  
5. **结果**：成功的图进「已发布」文件夹，失败的进「失败」文件夹，方便你排查。

技术上我们用了：**Node.js + TypeScript、Pinterest API v5、OpenAI 的看图接口、本地 OAuth、定时任务（node-cron）**，但对使用者来说，只需要会「放图、改配置、跑几条简单命令」就能完成从图片到 Pinterest Pin 的全流程。
